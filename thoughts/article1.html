# 【vue】使用v-for渲染的列表如何实现类的动态绑定？ juejin

# 需求描述

最近接到这样一个需求，觉得稍微有些复杂，想分析记录一下。这里简单描述要点。

页面上有选择按钮（button），其值有 button-val1 和 button-val2。还有一个选择框（select），后端返回选项值及该值是否存在禁用逻辑。 现在需要禁用选择框的某些选项，禁用条件是：button 的值为 button-val1，且选项值存在禁用逻辑。 也就是说，当button 选择button-val1时，select中select-val2被禁用；选择button-val2 时，select 中所有选项均可用。

# 细节分析

option 选项是由后端数据返回渲染的，所以必须使用 v-for 进行渲染，那么怎么在 v-for 中优雅地对每一个选项的样式进行差异化控制呢？ 首选关注禁用逻辑。分为两部分，一是由选项自身决定的，是否存在禁用逻辑，二是由前端按钮选项值决定的。 这里使用动态 class 绑定解决这个问题。class 绑定为对象，类是否启用，由上述两部分禁用逻辑共同构成。

# 代码实现

<template>
  <v-app>
    <v-row>
      <v-col cols="1">
        <div>button</div>
      </v-col>
      <v-col cols="2">
        <v-btn-toggle
          v-model="button_val"
          tile
          color="deep-purple accent-3"
          group
        >
          <v-btn value="button_val1">
            button_val1
          </v-btn>
          <v-btn value="button_val2">
            button_val2
          </v-btn>
        </v-btn-toggle>
      </v-col>
    </v-row>
    <v-row class="select"> 
      <v-col cols="1">
        <div class="select-res">option</div>
      </v-col>
      <v-col cols="2">
        <ul class="select-options">
          <li 
            v-for="obj in select_vals"
            :key="obj.val"
            :class="{'li-disabled':obj.disabled && disabled_option}"
          >{{obj.val}}</li>
        </ul>
      </v-col>
      </v-row>
  </v-app>
</template>
<script>

export default {
  name:'example',
  data(){
    return {
      select_vals: [{val:'option_val1',disabled:false},{val:'option_val2',disabled:true},{val:'option_val3',disabled:false}],
      button_val: 2,
    }
  },
  computed:{
    disabled_option(){
      return this.button_val === 'button_val1'
    }
  }
}
</script>
<style>
.li-disabled{
  background: saddlebrown;
}
</style>

# 实现效果

button选择button_val2，所有选项均可用

button选择button_val2，option_val2选项不可用
