# 关于 scoped 和 /deep/

vue 组件编译后，会将 template 中的每个元素加入 [data-v-xxxx] 属性来确保 style scoped 仅本组件的元素使用。 但如果引用了第三方组件，那默认只会对组件最外层的div加入这个属性，第二层开始就没有效果了。在 scoped 中修改引用组件内部的样式，是没有效果的。因为 scoped 中的css样式都会加上 [data-v-xxxx] 属性筛选。

way1：移除 scoped way2：深度选择器，对样式进行穿透 css:>>> less,scss:/deep/ 穿透方法实际上违反了 scoped 属性的意义。 way3：通过外层组件添加唯一class来区分组件+第三方样式来实现类似 scoped 的效果

# [TODO]【 ES6 】 weakmap 的使用

我们使用对象作为常规 Map 的键，那么当 Map 存在时，该对象也将存在。它会占用内存，并且不会被垃圾回收机制回收。

  • Map 对象的键可以是任何类型，但 WeakMap 对象中的键只能是对象引用；
  • WeakMap 不能包含无引用的对象，否则会被自动清除出集合（垃圾回收机制）；
  • WeakMap 对象是不可枚举的，无法获取集合的大小。

node --expose-gc map.js 命令执行 map.js 中的代码，其中 --expose-gc 参数表示允许手动执行垃圾回收机制。

function usageSize() {
  const used = process.memoryUsage().heapUsed;
  return Math.round((used / 1024 / 1024) * 100) / 100 + "M";
}

global.gc();
console.log(usageSize()); // ≈ 3.19M

let arr = new Array(10 * 1024 * 1024);
const map = new WeakMap();

map.set(arr, 1);
global.gc();
console.log(usageSize()); // ≈ 83.2M

arr = null;
global.gc();
console.log(usageSize()); // ≈ 3.2M

/**
 * 判断是否到达截止日期
 * @param dueDate 截止日期，格式：'2020-12-15'
 */
const dueDate = (dueDate) => {
  const today = new Date()
  const due = new Date(dueDate.replace(/-/g, '/'))
  return today >= due
}
